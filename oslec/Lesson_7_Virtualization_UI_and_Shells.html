Lesson 7: Virtualization and User Interface and Shells
7.1 المحاكاة الافتراضية (Virtualization)

المحاكاة الافتراضية هي تقنية بتسمح للـ OS إنه يقسم موارد الهاردوير بتاع جهاز واحد عشان يشغل أكتر من "كمبيوتر افتراضي" مستقل على نفس الجهاز. كل كمبيوتر افتراضي من دول بنسميه Virtual Machine (VM).

Operating systems and modern hardware provide a feature called virtualization that virtualizes the hardware such that each calling environment believes it has the dedicated access it needs to function. Virtualization is delivered via so-called virtual machines (VMs).

المكونات الأساسية للـ Virtualization:

  • Host OS: نظام التشغيل الأساسي اللي متسطب على الجهاز (زي ويندوز 10 عندك).
  • Guest OS: نظام التشغيل اللي شغال جوه الـ VM (ممكن يكون ويندوز 7، أو لينكس، أو أي نظام تاني).
  • Hypervisor: هو البرنامج الوسيط اللي بيدير الـ VMs وبيوزع عليهم موارد الهاردوير (CPU, RAM, ...). هو المدير بتاع البيئة الافتراضية.
• A VM has its own guest OS, which may be the same as or different from the underlying host OS. • The host OS provides a hypervisor, which manages the access to the hardware. • The user can completely customize their VM... without affecting the host OS or any other VM on the system.

مزايا الـ Virtualization:

ليه التقنية دي مهمة ومستخدمة بكثرة في السيرفرات والحوسبة السحابية؟

• Run operating systems where the physical hardware is unavailable. • Easier to create new machines, backup machines, etc. • Software testing using “clean” installs of operating systems and software. • Emulate more machines than are physically available. • Easy migration of virtual machines (shutdown needed or not). • Run legacy systems (تشغيل الأنظمة القديمة).
مثال عملي:
أنت مطور برامج وعندك جهاز عليه ويندوز 10 (Host OS). عايز تتأكد إن برنامجك شغال كويس على ويندوز 7 وعلى توزيعة لينكس اسمها Ubuntu. بدل ما تشتري جهازين كمان، ممكن تستخدم برنامج زي VMware (Hypervisor) عشان تعمل VM لويندوز 7 و VM تانية لـ Ubuntu (Guest OSs). هتقدر تشغلهم وتختبر برنامجك فيهم كأنهم أجهزة حقيقية، وكل ده على جهازك الأساسي.
7.2 الحماية (Protection)

الحماية هي مجموعة الآليات اللي الـ OS بيستخدمها عشان يتحكم في وصول البرامج والمستخدمين للموارد ويمنع أي ضرر. ده بيتم عن طريق خطوتين أساسيتين:

The most common protection would be to authenticate the requester and apply authorization to any new request to the system. • For example, when a request is made to a critical resource, the OS would verify the user request (which is called as authentication) and their approved access levels (which is called authorization) and controls before providing access.
  1. Authentication (التحقق من الهوية): دي الخطوة اللي الـ OS بيتأكد فيها "مين أنت؟". زي لما بتكتب اليوزرنيم والباسورد عشان تفتح الكمبيوتر.
  2. Authorization (منح الصلاحيات): بعد ما الـ OS عرف أنت مين، بيبص على الصلاحيات بتاعتك عشان يقرر "إيه اللي مسموحلك تعمله؟".

الـ OS بيستخدم حاجة اسمها Access Control Lists (ACLs) عشان يسجل فيها صلاحيات كل مستخدم على كل ملف أو مورد.

مثال:
لما تيجي تمسح ملف مهم في فولدر `C:\Windows`:
  • الـ OS بيسألك عن باسورد الأدمن (Authentication).
  • بعد ما تكتبه صح، الـ OS بيبص في الـ ACLs ويشوف هل الأدمن ده له صلاحية المسح في الفولدر ده ولا لأ (Authorization). لو له صلاحية، بيمسح الملف.
7.3 واجهة المستخدم والـ Shell

واجهة المستخدم (UI) هي الطريقة اللي بنتفاعل بيها مع نظام التشغيل. ورغم إنها مش جزء أساسي من الـ Kernel، لكنها جزء لا يتجزأ من تجربة استخدام الـ OS.

Although the user interface (UI) is not part of the OS kernel itself, this is typically considered to be an integral part of the OS. There can be multiple user interfaces... implemented either as a text-based interface or a graphical-based interface.

أنواع واجهات المستخدم:

  • Graphical User Interface (GUI): الواجهة الرسومية اللي بنستخدم فيها الماوس والأيقونات والنوافذ. سهلة للمستخدم العادي (زي سطح مكتب ويندوز).
  • Command Line Interface (CLI) / Shell: الواجهة النصية اللي بنكتب فيها أوامر. المبرمجين والـ System Admins بيفضلوها لأنها أسرع وأقوى في تنفيذ المهام المعقدة. (زي CMD أو PowerShell في ويندوز، و Bash في لينكس).
It is common for programmers to use the shell interface instead of the GUI for quickly traversing through the file system and interacting with the OS.
كمطور برامج، لازم تكون عارف إن الواجهة اللي المستخدم بيستخدمها (GUI أو CLI) بتأثر على طريقة كتابة برنامجك، زي طريقة التعامل مع الأوامر اللي بتجيلك من الـ command line أو طريقة عرض المخرجات.
