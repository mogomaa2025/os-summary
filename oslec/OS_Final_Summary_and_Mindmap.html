الملخص النهائي والخريطة الذهنية

مادة أنظمة التشغيل (Operating Systems)

ملخص شامل لأهم نقاط المادة (MCQ Focused)

الدرس 1: مقدمة

  • مكونات النظام: Hardware → OS → Applications → Users.
  • معمارية Von Neumann: الكود والبيانات في نفس الذاكرة.
  • مكونات الـ CPU: CU, ALU, Registers.
  • أهم Registers: PC (التعليمة الجاية)، CIR (التعليمة الحالية).
  • أهداف الـ OS: كفاءة، سهولة استخدام، تجريد.
  • أنواع الـ OS: Batch, Time Sharing, Real Time.

الدرس 2: العمليات والجدولة

  • Process: هو Program in Execution.
  • حالات العملية (5): New → Ready ↔ Running → Waiting → Terminated.
  • PCB: بطاقة هوية العملية (تحتوي على State, PC, Registers...).
  • Context Switch: حفظ حالة عملية وتحميل حالة عملية أخرى.
  • Thread: عملية خفيفة (Lightweight Process) لتحقيق التوازي داخل العملية الواحدة.

الدرس 3: إدارة الذاكرة

  • Address Binding: ترجمة العناوين من منطقية إلى فيزيائية.
  • Logical Address: العنوان اللي بيولده الـ CPU.
  • Physical Address: العنوان الحقيقي في الرامات.
  • MMU: هي الهاردوير المسؤول عن الترجمة.
  • IPC (التواصل): Shared Memory (سريع ومعقد) و Message Passing (أبطأ وآمن).

الدرس 4: إدارة I/O

  • Device Driver: برنامج وسيط بين الـ OS والجهاز.
  • أنواع الأجهزة: Block (زي الهارد) و Character (زي الكيبورد).
  • DMA: لنقل البيانات الكبيرة بدون إزعاج الـ CPU.
  • Interrupts: الطريقة الفعالة لإعلام الـ CPU بانتهاء عملية I/O.
  • Synchronization: لحماية البيانات المشتركة باستخدام Mutex (قفل) و Semaphore (عدّاد).
  • Deadlock: جمود يحدث عند تحقق 4 شروط (Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait).

الدرس 5: أنظمة الملفات

  • مكوناته: Directory Service (للتنظيم) و Storage Service (للتخزين الفعلي).
  • File Attributes: معلومات الملف (حجم، نوع، صلاحيات) وتخزن في FCB.
  • Directory Namespace: الهيكل الهرمي (الشجري) للفولدرات والملفات.
  • Concurrency Control: يمنع مسح أو تعديل ملف وهو مفتوح.

الدرس 6: الوصول والحماية

  • Protection Rings: مستويات صلاحيات لحماية النظام.
  • Ring 0 (Kernel Mode): أعلى صلاحية (للـ OS والـ Drivers).
  • Ring 3 (User Mode): أقل صلاحية (للبرامج العادية).
  • System Call: البوابة الآمنة للانتقال من User Mode إلى Kernel Mode لطلب خدمة.

الدرس 7: Virtualization & UI

  • Virtualization: تشغيل "كمبيوترات افتراضية" (VMs) على جهاز واحد.
  • مكوناتها: Host OS (الأساسي)، Guest OS (الضيف)، و Hypervisor (المدير).
  • الحماية: Authentication (مين أنت؟) و Authorization (مسموحلك تعمل إيه؟).
  • الواجهة (UI): GUI (رسومية) و CLI/Shell (نصية).
