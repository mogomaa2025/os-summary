ملخص مادة أنظمة التشغيل (OS)

الدرس الأول: مقدمة عن نظم الحاسوب ونظم التشغيل

Lesson 1: Introduction to Computer System and Operating System
1.1 مكونات نظام الكمبيوتر (Computer System Components)

أي نظام كمبيوتر بيتكون من 4 طبقات مترتبة فوق بعض بشكل هرمي، كل طبقة بتعتمد على اللي تحتها.

• Hardware: provides basic computing resources (CPU, memory, I/O devices) • Operating system: controls and coordinates the use of the hardware among the various application programs for the various users • Applications programs: define the ways in which the system resources are used to solve the computing problems of the users (compilers, database systems, video games, business programs) • Users: people, machines, other computers
مثال توضيحي:
تخيل إنك بتستخدم برنامج Word.
  • أنت (User) بتكتب على الكيبورد.
  • برنامج Word هو (Application Program) اللي بيستقبل الكتابة.
  • نظام التشغيل Windows هو (Operating System) اللي بياخد الكلام من البرنامج ويوصله للهاردوير.
  • البروسيسور والشاشة هما (Hardware) اللي بيعالجوا الكلام ويظهروه قدامك.
1.2 معمارية فون نيومان (Von Neumann Architecture)

ده التصميم الأساسي اللي اتبنى عليه معظم أجهزة الكمبيوتر في العالم من سنة 1945. أهم ميزة فيه هي فكرة "البرنامج المخزن".

• Von Neumann architecture is based on the stored-program computer concept, where instruction data and program data are stored in the same memory. This design is still used in most computers produced today. • His computer architecture design consists of a Control Unit (CU), Arithmetic and Logic Unit (ALU), Memory Unit, Registers and Inputs/Outputs.

1.2.1 وحدة المعالجة المركزية (CPU)

الـ CPU هو العقل المدبر للكمبيوتر، بيتكون من 3 أجزاء رئيسية:

• The CPU contains the ALU, CU and a variety of registers. • Registers are high speed storage areas in the CPU. All data must be stored in a register before it can be processed.
أهم المسجلات (Registers) ووظائفها:
  • MAR (Memory Address Register): بيخزن "عنوان" المكان في الذاكرة اللي عايزين نوصله.
  • MDR (Memory Data Register): بيخزن "البيانات" نفسها اللي بننقلها من أو إلى الذاكرة.
  • AC (Accumulator): بيخزن النتائج المؤقتة للعمليات الحسابية والمنطقية.
  • PC (Program Counter): بيخزن عنوان "التعليمة التالية" اللي عليها الدور تتنفذ.
  • CIR (Current Instruction Register): بيخزن "التعليمة الحالية" اللي بتتنفذ دلوقتي.

1.2.3 الناقلات (Buses)

دي عبارة عن الأسلاك أو المسارات اللي بتوصل كل مكونات الكمبيوتر ببعض.

• Address Bus: Carries the addresses of data (but not the data) between the processor and memory. • Data Bus: Carries data between the processor, the memory unit and the input/output devices. • Control Bus: Carries control signals/commands from the CPU... to control and coordinate all the activities.
1.3 نظام التشغيل (Operating System)

الـ OS هو برنامج بيدير الهاردوير وبيسهل تشغيل البرامج التانية. أول كود بيشتغل لما الكمبيوتر يفتح هو الـ bootstrap اللي بيكون متخزن في الـ ROM وبيحتوي على الـ BIOS.

An operating system, commonly referred to as the OS, is a program that controls the execution of other programs running on the system. It acts as a facilitator and intermediate layer between the different software components and the computer hardware.
أهداف أي OS:
  • Efficiency: الكفاءة والسرعة.
  • Ease of usability: سهولة الاستخدام للمستخدم.
  • Ability to abstract and extend: القدرة على إخفاء تعقيدات الهاردوير وإضافة أجهزة جديدة بسهولة.
أجزاء الـ OS:
  • Kernel (النواة): الجزء الأساسي اللي شغال طول الوقت في الذاكرة وبيتعامل مع الهاردوير مباشرة.
  • Libraries, Applications, and Tools: البرامج والأدوات المساعدة اللي بتيجي مع الـ OS.
1.3.2 فئات أنظمة التشغيل (OS Categories)

بنقسم أنظمة التشغيل بطريقتين:

1. حسب نوع الاستخدام (Usage Types):
  • Batch: بينفذ مجموعة مهام ورا بعض من غير تدخل بشري. (زي أنظمة الشركات الكبيرة زمان).
  • Time Sharing: بيقسم وقت الـ CPU بين كذا مستخدم أو برنامج عشان الكل يحس إنه شغال في نفس الوقت. (زي Windows و Mac).
  • Parallel – Distributed: بيشغل أجهزة كتير كأنهم جهاز واحد قوي.
  • Network: بيربط أجهزة مختلفة عن طريق شبكة.
  • Real Time: بيحتاج دقة واستجابة في أجزاء من الثانية. (زي أنظمة التحكم في الطيارات والمصانع).
2. حسب التصميم (Designed and Supported Features):
  • Monolithic: الـ OS كله عبارة عن كتلة واحدة في الـ Kernel. لو حصلت مشكلة في جزء، النظام كله ممكن يتأثر.
  • Modular: الـ OS متقسم لوحدات (Modules) زي تعريفات الأجهزة، وتقدر تحدث كل جزء لوحده. (معظم الأنظمة الحديثة).
  • Micro-service based: أحدث تصميم، الـ OS متقسم لخدمات صغيرة جداً ومستقلة، وده بيسهل الصيانة والتطوير.
1.4 لماذا نحتاج نظام تشغيل؟ (Why We Need an OS?)

ببساطة، بنحتاجه عشان:

  • يشغل البرامج ويسهل عليها استخدام الهاردوير.
  • يدير التعارضات بين البرامج اللي عايزة تستخدم نفس الموارد في نفس الوقت.
  • يخفي تعقيدات الهاردوير عن المبرمج والمستخدم.
الأنظمة المعقدة ومتعددة المعالجات (Complex and Multiprocessor Systems)

الأجهزة الحديثة فيها بروسيسورات بكذا نواة (Core).

• When all cores provide the same or identical capabilities, they are called as homogeneous platforms. • There could also be systems that provide different capabilities on different CPU cores. These are called heterogeneous platforms.

الـ OS لازم يكون ذكي كفاية عشان يوزع الشغل صح على الأنوية المختلفة دي (efficient scheduling).

تعدد المهام (Multitasking) وتعدد المستخدمين (Multiuser)

الـ OS بيسمح بتشغيل برامج كتير في نفس الوقت (برامج في الواجهة foreground وبرامج في الخلفية background). كمان بيسمح لكذا مستخدم (زي الـ Admin والـ User العادي) إنهم يستخدموا نفس الجهاز بصلاحيات مختلفة.

1.7 مسؤوليات نظام التشغيل (Responsibilities of an OS)

الـ OS عنده 6 مسؤوليات أساسية، وكل درس من الدروس الجاية هيشرح واحدة منهم بالتفصيل:

They are broadly classified into the following functional areas: • Scheduling • Memory management • I/O and resource management • Access and protection • File systems • User interface/shell
