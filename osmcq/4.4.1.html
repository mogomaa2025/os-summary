What is another name for Special Instruction I/O?
Explanation
"Special Instruction I/O or Port-mapped IO (PMIO)"
Source: Page 66, Section 4.4.1
2
How does Special Instruction I/O work?
Explanation
"There could be specific CPU instructions that are custom developed for communicating with and controlling the I/O devices."
Source: Page 66, Section 4.4.1
3
According to the diagram, how does the CPU differentiate between accessing memory and an I/O device?
Explanation
The diagram on page 67 shows the IORQ (I/O Request) control line. When IORQ=0, memory is accessed. When IORQ=1, the I/O address space is accessed. The text in the diagram says "The IORQ control line indicate which address space the CPU want to access."
Source: Page 67, Section 4.4.1
4
What is a primary reason for using Special Instruction I/O?
Explanation
"This may be needed for faster and efficient communication."
Source: Page 66, Section 4.4.1
5
In a system using Port-mapped I/O, what is true about the memory and I/O address spaces?
Explanation
The diagram on page 67 explicitly states, "Memory and I/O devices have their own address space." This separation is the key principle of Port-mapped I/O.
Source: Page 67, Section 4.4.1
6
True or False: Special Instruction I/O is the most common type of I/O protocol and is used for the majority of devices. TRUE/FALSE
Explanation
This is false. The text notes, "However, such type of I/Os are special and smaller in number."
Source: Page 66, Section 4.4.1
